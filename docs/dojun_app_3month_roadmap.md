# 전체 전략

핵심 원칙

1.  밴드를 **즉시 종료하지 않는다**
2.  **앱이 더 편리하다는 경험**을 먼저 만든다
3.  **공지 → 참여 → 자료 → 알림** 순으로 확장한다

전환 전략

BAND 병행 → BAND 의존 감소 → BAND 종료

# 1개월차 : 기반 안정화 (기술 + 구조)

목표 - 앱 기능 안정화 - 공지 전달 시스템 확립 - 관리자 운영 규칙 확정

## 개발 작업

### 모바일 앱 안정화

-   Firestore 데이터 동기화 검증
-   공지 / 투표 / 설문 표시 안정화
-   이미지 공지 표시 테스트

### 강제 업데이트 시스템

-   meta/version 문서 기반 업데이트 체크
-   구버전 차단 기능 테스트

### 데이터 Refresh 구조

-   앱 실행 시 자동 refresh
-   메인 화면 진입 시 refresh
-   수동 refresh 버튼 유지

------------------------------------------------------------------------

## 관리자 운영 규칙 확정

문서 상태 규칙

draft → 작성 중\
review → 내부 검토\
open → 조합원 공개\
closed → 종료\
archived → 보관

모바일 노출 규칙

공지 - open - closed

투표 / 설문 - open
